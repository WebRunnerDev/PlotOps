# Board-scoped Sprints with commitment at Start

Sprints belong to a Board (not Project/Team): timebox + Task container only — columns stay on the Board. At most one Active Sprint per Board; many Drafts allowed. Start snapshots Commitment; later add/remove on Active are Scope changes. Completion is decided only in the Close dialog (pre-suggest last column); Cancel returns all Tasks to the Backlog. Manager+ owns lifecycle/membership; Contributor views. No story points in MVP (count-based metrics).
