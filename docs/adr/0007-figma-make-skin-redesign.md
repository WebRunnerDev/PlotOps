# Figma Make redesign is a skin, not a feature rewrite

When porting [Dark-themed CRM Interface Design](https://www.figma.com/make/qf1x9Yy4S0wbZJ8IgrE2qS/Dark-themed-CRM-Interface-Design) into PlotOps, keep existing feature structure and navigation (Board switcher, projects list exit, current Task drawer fields/sections). Apply Make as visual skin plus local layout within those slots (tokens, typography, spacing, card/column/header hierarchy). Do not drop or invent product behavior to match Make gaps or extras — Make-only UI without a code counterpart goes to a deferred backlog, not the redesign PR.

First pass scope: Board shell → kanban columns/cards → Task drawer. Start by aligning global tokens in `src/app/styles/index.css`, then tune those surfaces; avoid per-component hardcoded hex when a token exists. Keep PlotOps font families (Space Grotesk / IBM Plex Sans / JetBrains Mono) — Make already matches them. Make-only UI without a code counterpart is listed in `docs/SPEC.md` → **Deferred from Figma Make** (separate table, not mixed with other deferred product items). Task drawer: copy Make visual language only (density, section labels, mono meta, colors); keep our existing content/sections — do not adopt Make’s two-column field set. Board header: keep PlotOps controls (back to projects, Board switcher, archive, settings, GitHub, existing filters); restyle only — do not add Make chrome for missing features. App dock: keep current items (home / notifications / settings / account); restyle chrome only — Make Board/CI/CD/Branches IA can be added later if product needs it.
